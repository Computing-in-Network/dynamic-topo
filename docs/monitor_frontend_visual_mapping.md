# Monitor 数据到前端展示映射（V1）

## 1. 设计目标
- 一眼看懂当前网络健康状态。
- 告警可快速定位到拓扑对象。
- 指标和颜色语义统一，不依赖口头解释。

## 2. 页面分层
- 总览层：健康分、告警计数、关键 KPI。
- 拓扑层：节点与链路状态可视化、路径高亮、异常强调。
- 诊断层：对象详情、趋势、最近事件。
- 时间层：告警时间线与状态变化回看。

## 3. 事件类型与组件映射

| 事件 | 关键字段 | 主展示组件 | 视觉规则 |
|---|---|---|---|
| `node_metric` | `node_id`, `cpu_ratio`, `mem_ratio`, `conn_count`, `tx_bps`, `rx_bps`, `status` | 节点样式 + 节点详情抽屉 | 节点外环按状态上色，详情显示数值与单位 |
| `link_metric` | `link_id`, `src_node_id`, `dst_node_id`, `state`, `loss_rate`, `rtt_ms`, `jitter_ms` | 链路线条 + 链路详情卡片 | 线条颜色表达健康度，粗细表达负载 |
| `flow` | `flow_id`, `src_node_id`, `dst_node_id`, `path`, `bps`, `priority` | 路径高亮层 + 流详情面板 | 路径动态高亮，优先级影响亮度或描边 |
| `alarm` | `alarm_id`, `severity`, `scope_type`, `scope_id`, `title`, `detail` | 告警列表 + 时间线 | 严重级别颜色固定，点击后联动定位 |

## 4. 状态与阈值规则（默认）

| 指标 | normal | warning | critical |
|---|---|---|---|
| `cpu_ratio` | < 0.70 | 0.70 ~ 0.90 | > 0.90 |
| `mem_ratio` | < 0.75 | 0.75 ~ 0.90 | > 0.90 |
| `loss_rate` | < 0.01 | 0.01 ~ 0.05 | > 0.05 |
| `rtt_ms` | < 80 | 80 ~ 150 | > 150 |
| `jitter_ms` | < 20 | 20 ~ 50 | > 50 |

说明：以上是 V1 默认值，最终以业务配置为准，支持后续参数化。

## 5. 颜色语义（固定）
- normal：绿色系（可用、健康）。
- warning：黄色系（退化、需关注）。
- critical：红色系（异常、需处置）。
- unknown：灰色系（无数据或字段缺失）。

## 6. 交互联动规则
1. 点击告警行，自动定位到 `scope_id` 对应节点或链路。
2. 点击节点或链路，右侧诊断面板显示最近指标与关联告警。
3. 选择流量记录时，拓扑中高亮 `path`，并弱化非相关链路。
4. 支持“仅异常对象”筛选，减少信息噪声。

## 7. 缺失字段降级策略
- 缺失可选字段时显示 `--`，并保持组件可渲染。
- 缺失关键标识字段（如 `node_id`、`link_id`）时丢弃该条并记录日志。
- `timestamp` 缺失时使用接收时间，并标记为 `server_filled`。

## 9. 验收场景（直观性）
1. 注入高丢包链路后，链路在拓扑中 1 秒内转为异常色，告警区出现对应条目。
2. 点击该告警后，自动定位到目标链路并展开详情。
3. 节点 CPU 持续升高到临界值时，节点颜色从 normal 到 warning 再到 critical。
4. 在字段缺失情况下，页面不崩溃且缺失信息可识别。
